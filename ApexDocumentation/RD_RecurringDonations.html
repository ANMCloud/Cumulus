RD_RecurringDonations

public with sharing class RD_RecurringDonations
The main class for controlling the behavior of Recurring Donations, both open ended and standard. The logic of this class is quite complicated, so here is a summary of the call chain for various scenarios. Insert a new RD (open or non open): RD_RecurringDonations_TDTM(AfterInsert) insertOppsOnRecurringDonationInsert Database.insert(opps) updateRecurringDonationOnOppChange Update a non open-ended RD: RD_RecurringDonations_TDTM(AfterUpdate) handleRecurringDonationUpdate delete non-closed Opps insertOppsOnRecurringDonationInsert Update a open-ended RD to closed: RD_RecurringDonations_TDTM(AfterUpdate) handleRecurringDonationUpdate runRecurringDonationClosedActions Update an open-ended RD with changes that require opp updating: RD_RecurringDonations_TDTM(AfterUpdate) handleRecurringDonationUpdate updateExistingOpps updateRecurringDonationOnOppChange Update an open-ended RD to different schedule or contact (need to recreate opps): RD_RecurringDonations_TDTM(AfterUpdate) handleRecurringDonationUpdate delete non-closed Opps insertOppsOnRecurringDonationInsert Refresh Opportunities button or Nightly Batch on non open-ended: evaluateRecurringDonationsForNewOppInsert delete non-closed Opps insertOppsOnRecurringDonationInsert Refresh Opportunities button or Nightly Batch on open-ended: evaluateRecurringDonationsForNewOppInsert database.insert(opps) (for additional opps) insertOppsOnRecurringDonationInsert (if no opps exist) updateRecurringDonationOnOppChange Update an Opportunity that belongs to a RD: RD_RecurringDonations_Opp_TDTM(AfterUpdate) updateRecurringDonationOnOppChange

Methods

  • approveDeleteRecDonations
    Validates that the Recurring Donations can be deleted, and marks the RD record with an error if there are any Opportunities referring to the RD.
  • dtEndOfMonthFixup
    Returns the last day of month for the provided date, assuming is set for the 28th or later. This is a way to avoid the last day of month getting pushed to the 30th, and then the 28th.
  • evaluateRecurringDonationsForNewOppInsert
    Handles changes to the specified RD's to update/recreate their Opportunities. Called from the Refresh Opportunities button on both Open and Non-Open RD's, as well as the Batch process for extending Open Ended RD's (no Non-Open).
  • getNextDate
    Returns the next date in the sequence based on the RD schedule type
  • handleRecurringDonationUpdate
    Trigger AfterUpdate Handler for updates on existing Recurring Donations. Key scenarios include: - closing an open ended rd, which should close all remaining opps on the rd. - opening a closed rd, which should create new opps on the rd. - any other modification to the rd, which should update the rd's opps.
  • insertOppsOnRecurringDonationInsert
    Creates new Opportunities for the specified Recurring Donations. Called by the AfterInsert Trigger Handler on RD's, and also when RD's are updated and we delete and recreate their Opps.
  • insertOppsOnRecurringDonationInsertFuture
    Future method that creates new Opportunities for the specified Recurring Donations.
  • oppContactRoles
    Creates household OpportunityContactRoles for the specified Opps. Since NPSP3, the RD code always leverages opp.npe01__Contact_Id_for_Role__c and this code isn't used.
  • oppContactRolesFuture
    Future method that creates household OpportunityContactRoles for the specified Opps. Since NPSP3, the RD code always leverages opp.npe01__Contact_Id_for_Role__c and this code isn't used.
  • requeryListRD
    Returns a list of RD's for the specified Id's, with all their fields queried.
  • runRecurringDonationClosedActions
    When closing an RD, either deletes or closes all of its currently open Opps.
  • strQueryRDNoWhere
    Returns the soql string for RD's. Includes fields from the custom field map and currency iso fields. Does not include the where clause since different callers have different filtering needs.
  • updateExistingOpps
    Updates all the existing Opportunities for the specified Recurring Donations.
  • updateExistingOppsFuture
    Future method that updates all the existing Opportunities for the specified Recurring Donations.
  • updateRecurringDonationOnOppChange
    Updates the specified RD's by querying their Opps so it can update the RD's Next Payment Date, Last Payment Date, Paid Amount, Number of Paid Installments.
  • updateRecurringDonationOnOppChangeFuture
    Future method that updates the specified RD's by querying their Opps so it can update the RD's Next Payment Date, Last Payment Date, Paid Amount, Number of Paid Installments.
